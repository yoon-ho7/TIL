~~~
Github란
-깃을 사용하는 프로젝트를 지원하는 서비스
~~~

~~~
버전관리란
-동일한 정보에 대한 여러 버전 관리 쓰는 이유
-협업에 필요
-코드를 공유하면 개발 파트를 나눔
-같은 기능을 개발해도 공유하면 같이 개발 가능
~~~

~~~
Git쓰는 이유
-코드를 주고 받을 필요 없이 같은 파일을 여러 명이 동시에 작업 가능
-인터넷 연결 없이 개발 가능,복구 가능
~~~

~~~
Branch
-독립적으로 어떤 작업을 진행하기 위한 개념
-각각의 브랜치는 다른 브랜치의 영향을 받지 않음
-여러 작업을 동시에 진행 가능
~~~

~~~
Merge
-브랜치를 다른 브랜치와 병합하는 작업
-작업한 내용을 새로운 하나의 브랜치로
~~~

~~~
Pull request : 완료한 작업을 다른 사람이 리뷰하고 병합하도록 요청
Push : 내 컴퓨터 로컬에 저장돼있던 버전 정보를 git저장소에 올리기
Pull : Git 저장소 서버로부터 내 컴퓨터 로컬로 버전 정보 전체를 가져오기
~~~
